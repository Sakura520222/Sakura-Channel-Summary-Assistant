## 常见代码问题和审查要点
- 布尔配置陷阱：`bool('false')==True`，必须类型感知解析；禁止使用`bool(value)`作为未经检查的兜底。
- 版本号DRY：版本号必须在且仅在`core/__init__.py`中定义，禁止其他文件硬编码。CI应加入`grep -r --exclude=core/__init__.py '"1\.[0-9]+\.[0-9]+"' core/`检查。
- 文件I/O安全：禁止全量读取未知大小文件（如`Path.read_text()`），必须使用流式/尾部读取；API响应禁止暴露服务器文件路径，须脱敏。
- 竞态条件预防：对文件系统进行“检查-读-改-写”时，禁止使用`exists()`预检查，必须用`try/except`捕获`FileNotFoundError`。
- 整数转换异常：从请求提取整数参数时，必须`try/except ValueError`捕获，返回400而非500。
- 异步资源清理：`aiofiles.open`必须使用`async with`上下文管理器；`asyncio.create_task`必须记录Task并在shutdown时取消。
- 审计日志完整性：所有写操作必须记录审计，失败时`success=False`也须写入。
- 删除接口语义一致性：Web API与Telegram命令的删除接口语义宜保持一致，不一致须在PR描述中说明。
- 类型安全：`type: ignore`须追踪原因，判断是否可改进类型注解。

## 规范建议和经验教训
- **文件句柄泄露检查**：所有`open()`（含`aiofiles.open`）必须使用上下文管理器。
- **大文件读取策略**：`deque(maxlen=N)`是处理大文件的好模式，过滤后行数不足时应附带`warning`或`truncated`标志。
- **版本号硬编码CI门禁**：项目增加静态检查阻止硬编码。
- **审计日志模式**：应强制包裹try/except，成功和失败路径都记录。
- **配置重载并发**：低频操作可用文档声明“不支持并发调用”而非加锁。
- **异步协程检测**：使用`inspect.isawaitable`而非`hasattr(obj, "__await__")`。
- **经验系数可溯源**：魔数（如`lines*3`）须注释说明推导过程。
- **依赖同步检查**：建议使用`aiofiles`时须验证`requirements.txt`已添加。
- **测试覆盖强制**：新增API须包含正常、异常、边界（空body）用例。
