### 二、发现的模式

#### 代码模式
1. **Naive/Aware Datetime 混用反模式**（项目记忆 #7）
   - 现状：`datetime.now(UTC)` 生成 aware，直接传入 naive-only 的 MySQL DATETIME
   - 本次修复：手动 `.replace(tzinfo=None)` 打补丁
   - 风险：系统性遗漏（24 处未修复）

2. **防御性编程的过度使用**
   - `version_utils.py` 的 `build_frontend()` 中逐层 `if` 检查 + 延迟导入，但核心的 `subprocess.run()` 异常处理反而简单（只检查 returncode）
   - 体现：倾向于“检查前置”而非“异常捕获”，可能掩盖真实错误

3. **跨语言逻辑重复**
   - Shell 脚本 + Python 各自实现前端构建，且未交叉引用
   - 反映：缺少统一的“操作编排层”（如 Makefile、just、任务队列）

#### 架构观察
1. **数据库抽象层泄露**（项目记忆提醒）
   - `submission_repo.py` 直接构造 datetime 字符串参数 → 绕过 `mysql.py` 的转换层
   - 修复仅停在 Repo 层，未向下抽象到 Repository 基类或 Model 层

2. **迁移脚本设计哲学矛盾**
   - 期望：“自动执行、不阻塞启动”
   - 现实：`database_initializer.py` 中调用迁移 → 若数据库锁超时，整个应用启动失败
   - 应建议：异步后台执行 + 健康检查跳过未完成迁移

3. **版本管理巧妙设计**
   - 优先读 `core/__init__.py`，回退 `main.py`，实现平滑迁移
   - 但未解决：两个文件版本不一致时如何决策？应记录警告

### 三、规范完善建议

#### 建议新增审查规则

| 规则 ID | 名称 | 描述 | 严重程度 |
|---------|------|------|----------|
| SR-007 | Datetime 存储统一 | 写入 MySQL DATETIME 必须使用 naive UTC datetime，禁止 aware datetime 传入 | 🟡 |
| SR-008 | 迁移脚本幂等性 | 所有迁移必须检查 `alembic_version` 或自定义标记表，避免重复执行 | 🟡 |
| SR-009 | 同步 I/O 在异步上下文 | 异步函数中禁止直接使用 `open()`、`time.sleep()` 等阻塞调用 | 🔴 |
| SR-010 | 跨语言逻辑单点 | Shell 与 Python 重复的逻辑必须明确主实现，另一处通过调用或注释引用 | 🟢 |

#### 项目记忆补充
```markdown
- **Datetime 存储契约**：所有写入 MySQL DATETIME 的值必须通过 `mysql.py` 的统一转换函数 `to_db_datetime(dt: datetime | None) -> str | None` 处理，禁止在 Repo 层直接传参。
- **迁移脚本设计**：迁移必须在后台异步执行，启动时检查是否完成；若未完成，应用应正常运行但禁用相关功能（如投稿审核），并在日志中持续提醒。
- **异步上下文文件 I/O**：所有文件读取必须使用 `aiofiles`，或确保在 `executor` 中运行同步代码；禁止直接 `open()`。
```

### 四、经验教训

#### 未来审查关注点
1. **系统性问题的局部修复**：当发现一个模式性错误（如 aware datetime）只修了 1/25 处时，必须：
   - 标记为“系统性风险 + 局部修复”
   - 建议创建技术债 Issue 跟踪剩余 24 处
   - 在 PR 中要求至少修复 80% 或添加 TODO 注释

2. **Shell ↔ Python 集成边界**：审查时必须交叉验证：
   - Python 调用的 shell 命令与手动执行的 shell 脚本是否行为一致？
   - 错误处理和返回值传播是否正确？

3. **迁移脚本的双刃剑**：
   - 优点：自动修复数据
   - 缺点：启动阻塞、无回滚、难以调试

- **时区混乱债务**：24 处 aware datetime 写入点 → 需整体重构 `mysql.py` 的 `_execute` 方法，自动转换 aware → naive

- **迁移脚本债务**：没有版本管理表 → 建议引入 `alembic` 或自建 `schema_migrations` 表

- ✅ 跨文件模式扫描：发现 `mysql.py` 中 24 处同类问题
